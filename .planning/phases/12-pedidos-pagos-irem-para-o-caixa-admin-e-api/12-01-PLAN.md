# Phase 12: Pedidos pagos irem para o caixa admin e api - Plan

## Workflow Overview
Implement the Cash Register (Caixa) feature to track paid orders within specific time periods.

## File Modifications

### 1. Database Schema and Push
- **File**: `prisma/schema.prisma`
- **Action**: Add `CashRegister` model and `paymentDate DateTime?` to `Order`.
- **Command**: `npx prisma db push --accept-data-loss` (requires manual/blocking execution)

### 2. Update Orders Use Cases
- **File**: `src/modules/orders/domain/use-cases/receive-order.use-case.ts`
- **Action**: Set `paymentDate = new Date()` on payment.
- **File**: `src/modules/orders/domain/use-cases/revert-receive-order.use-case.ts`
- **Action**: Set `paymentDate = null` on revert.

### 3. API - Cash Registers Module
- **Files**:
  - `src/modules/cash-registers/domain/entities/cash-register.entity.ts`
  - `src/modules/cash-registers/infrastructure/database/prisma-cash-registers.repository.ts`
  - `src/modules/cash-registers/domain/use-cases/create-cash-register.use-case.ts`
  - `src/modules/cash-registers/domain/use-cases/list-cash-registers.use-case.ts`
  - `src/modules/cash-registers/domain/use-cases/get-cash-register-summary.use-case.ts`
  - `src/modules/cash-registers/infrastructure/controllers/cash-registers.controller.ts`
  - `src/modules/cash-registers/cash-registers.module.ts`
- **Action**: Build CRUD and summary use cases for `CashRegister`. Add module to `app.module.ts`.

### 4. Frontend - Cash Registers Service
- **File**: `../ecommerce-admin-front/src/services/cash-register.service.ts`
- **Action**: Create Axios service to interact with `/caixas` endpoints.

### 5. Frontend - UI and Routes
- **Files**:
  - `../ecommerce-admin-front/src/pages/CashRegistersPage.tsx`
  - `../ecommerce-admin-front/src/pages/CashRegisterDetailsPage.tsx`

- **Action**: Build the list and details UI. Add routes.
