# Plano 07-01: Extensão do Schema, DTO e Repositório da API para Módulo de Pedidos (Backend)

Este plano detalha o desenvolvimento do módulo de Pedidos (`orders`) no backend (`ecommerce-core-api`) utilizando a Clean Architecture com NestJS e Prisma.

---

## 🎯 Objetivo

* **Modelagem de Dados**: Criar tabelas robustas `Order` e `OrderItem` no PostgreSQL para guardar informações detalhadas de clientes, itens comprados, variações selecionadas, precificações, frete, descontos e endereços de entrega.
* **Módulo Clean Architecture**: Desenvolver a estrutura desacoplada em domínio (`domain`) e infraestrutura (`infrastructure`), expondo rotas REST seguras e testadas para o admin.

---

## 🛠️ Mudanças Propostas

### 1. Banco de Dados (Prisma)

#### [MODIFY] [schema.prisma](file:///c:/sites/podemais/ecommerce-api/prisma/schema.prisma)
* Criar as entidades `Order` e `OrderItem` com enums de status adequados:
  ```prisma
  enum OrderStatus {
    PENDING
    CONFIRMED
    DISPATCHED
    COMPLETED
    CANCELLED
  }

  model Order {
    id              String      @id @default(uuid())
    orderNumber     Int         @unique @default(autoincrement())
    customerName    String
    customerPhone   String
    
    // Valores monetários
    itemsTotal      Decimal     @db.Decimal(10, 2)
    freight         Decimal     @db.Decimal(10, 2)
    discount        Decimal     @db.Decimal(10, 2)
    totalOrder      Decimal     @db.Decimal(10, 2)
    totalReceived   Decimal     @db.Decimal(10, 2)

    // Pagamento
    paymentType     String      // Ex: "Online", "Presencial"
    paymentMethod   String      // Ex: "PIX", "Cartão de Crédito", "Cartão de Débito", "Dinheiro"
    pixKey          String?

    // Endereço de Entrega
    street          String
    number          String
    neighborhood    String
    city            String
    state           String
    cep             String
    complement      String?

    status          OrderStatus @default(PENDING)
    
    createdAt       DateTime    @default(now())
    updatedAt       DateTime    @updatedAt

    items           OrderItem[]

    @@map("orders")
  }

  model OrderItem {
    id              String   @id @default(uuid())
    orderId         String
    productName     String
    price           Decimal  @db.Decimal(10, 2)
    quantity        Int
    variation       String?  // Armazena a string descritiva da variação escolhida
    imageUrl        String?

    order           Order    @relation(fields: [orderId], references: [id], onDelete: Cascade)

    @@map("order_items")
  }
  ```

### 2. Camada de Domínio (`src/modules/orders/domain`)

#### [NEW] `domain/entities/order.entity.ts`
* Criar a classe/entidade pura de domínio representando o pedido, com regras de negócio como validação e o método utilitário `cancel()` que atualiza o status para `CANCELLED`.

#### [NEW] `domain/repositories/iorders.repository.ts`
* Criar a classe abstrata `IOrdersRepository` definindo o contrato de persistência:
  * `abstract findMany(filters: OrderFilters): Promise<Order[]>`
  * `abstract findById(id: string): Promise<Order | null>`
  * `abstract updateStatus(id: string, status: OrderStatus): Promise<void>`
  * `abstract save(order: Order): Promise<Order>`

#### [NEW] `domain/use-cases/`
* `list-orders.use-case.ts`: Consulta pedidos com suporte a filtros dinâmicos de texto, status, e agrupamento/ordenação temporal (do mais novo para o mais antigo).
* `get-order-detail.use-case.ts`: Retorna o pedido detalhado completo com seus itens vinculados por ID.
* `cancel-order.use-case.ts`: Modifica o status do pedido para `CANCELLED` executando a validação de domínio e persistindo no repositório.

---

### 3. Camada de Infraestrutura (`src/modules/orders/infrastructure`)

#### [NEW] `infrastructure/database/prisma-orders.repository.ts`
* Implementar o repositório Prisma convertendo modelos Prisma para entidades do domínio e realizando as persistências corretas das tabelas `orders` e `order_items`.

#### [NEW] `infrastructure/dtos/`
* `query-orders.dto.ts`: Validações de queries de filtros (search, status, date) via `class-validator` e Swagger `@ApiProperty`.
* `update-order-status.dto.ts`: Valida o enum de novos status passados no body.

#### [NEW] `infrastructure/controllers/orders.controller.ts`
* Expor as seguintes rotas Rest com decorators `@UseGuards(JwtAuthGuard)` de administrador:
  * `GET /orders` -> `listOrdersUseCase.execute(query)`
  * `GET /orders/:id` -> `getOrderDetailUseCase.execute(id)`
  * `POST /orders/:id/cancel` -> `cancelOrderUseCase.execute(id)`

#### [NEW] `orders.module.ts`
* Declarar o módulo NestJS mapeando os Use Cases e provendo `IOrdersRepository` através da classe concreta do Prisma. Registrar o módulo no `src/app.module.ts`.
