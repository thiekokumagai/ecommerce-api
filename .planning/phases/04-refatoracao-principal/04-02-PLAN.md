# Phase 4 Plan: 04-02 - Refatorar módulo de autenticação (auth)

## Objective
Desacoplar a lógica de negócio do módulo de `auth`, extraindo a validação de credenciais e a geração de JWT para Use Cases no diretório `domain`. Manter os recursos técnicos (Guards e Passport) isolados no diretório `infrastructure`.

## Tasks

### 1. Criar estrutura do domínio (auth)
- [ ] Criar diretório `src/modules/auth/domain/use-cases`
- [ ] Criar interfaces e contratos necessários.

### 2. Migrar Lógica para Use Cases (auth)
- [ ] Criar `ValidateUserUseCase` (verifica o e-mail, busca usuário e valida o hash da senha).
- [ ] Criar `LoginUseCase` ou `GenerateTokenUseCase` (responsável por encapsular o payload e criar o JWT).
- [ ] Isolar as lógicas que antes estavam em `auth.service.ts`.

### 3. Atualizar Infraestrutura (auth)
- [ ] Atualizar `auth.controller.ts` para injetar os Use Cases de domínio.
- [ ] Garantir que o `JwtStrategy`, `LocalStrategy` e `JwtAuthGuard` fiquem na pasta `infrastructure` e funcionem adequadamente chamando os Use Cases em vez de services acoplados.

### 4. Limpeza e Testes
- [ ] Excluir o antigo `auth.service.ts`.
- [ ] Atualizar `AuthModule` para prover Use Cases e os componentes do Passport.
- [ ] Rodar testes de login/acesso e garantir que a autenticação no sistema permaneça consistente e intacta.
