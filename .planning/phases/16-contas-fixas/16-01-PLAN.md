# Plano 16-01: Cadastro de Custos Fixos, Lançamento de Saídas e Conciliação de Caixa (Backend)

Este plano descreve o fluxo técnico de desenvolvimento no backend para suportar o cadastro e gerenciamento de custos fixos ("Contas fixas") e o registro de movimentações de caixa manuais (entradas e saídas de despesas). As saídas de custos fixos serão liquidadas contra o caixa aberto atual, debitando-as do saldo líquido consolidado.

---

## 🎯 Objetivos do Desenvolvimento

1. **Modelagem de Dados**: Criar as tabelas `fixed_costs` (cadastro de custos fixos) e `cash_transactions` (movimentações financeiras de caixa) no banco de dados via Prisma.
2. **Arquitetura Limpa Modular**: Implementar o módulo de custos fixos seguindo o padrão modular sob `src/modules/fixed-costs`.
3. **Casos de Uso Principais**:
   - CRUD completo de Custos Fixos (`FixedCost`).
   - Lançamento de Baixa de Custo Fixo (`PayFixedCostUseCase`), gerando um registro em `CashTransaction` do tipo `"OUTFLOW"` (saída) vinculado ao caixa ativo atual.
   - Suporte a lançamentos manuais adicionais de entrada (`ENTRY`) ou saída (`OUTFLOW`) de caixa.
4. **Consolidação Financeira de Caixa**: Atualizar o `GetCashRegisterSummaryUseCase` para computar entradas (`totalEntries`), saídas (`totalOutflows`), reajustar faturamento bruto, taxas e saldo líquido total, incluindo as movimentações no extrato de retorno.

## 🛠️ Detalhamento Técnico das Mudanças

### Onda 1: Esquema de Banco e Migração (Prisma)

#### 1. Atualização do Schema Prisma
- **Arquivo**: `prisma/schema.prisma`
- **Modificações**:
  - Atualizar o model `CashRegister` para adicionar a relação opcional:
    ```prisma
    transactions CashTransaction[]
    ```
  - Adicionar o model `FixedCost`:
    ```prisma
    model FixedCost {
      id                String            @id @default(uuid())
      name              String
      value             Decimal           @db.Decimal(10, 2)
      repeats           Boolean           @default(true)
      type              String            @default("ALWAYS") // "ALWAYS", "INSTALLMENTS"
      installmentsCount Int?              @default(1)
      createdAt         DateTime          @default(now())
      updatedAt         DateTime          @updatedAt
      transactions      CashTransaction[]

      @@map("fixed_costs")
    }
    ```
  - Adicionar o model `CashTransaction`:
    ```prisma
    model CashTransaction {
      id             String        @id @default(uuid())
      cashRegisterId String?
      type           String        // "ENTRY", "OUTFLOW"
      amount         Decimal       @db.Decimal(10, 2)
      description    String
      date           DateTime      @default(now())
      fixedCostId    String?
      createdAt      DateTime      @default(now())

      cashRegister   CashRegister? @relation(fields: [cashRegisterId], references: [id], onDelete: SetNull)
      fixedCost      FixedCost?    @relation(fields: [fixedCostId], references: [id], onDelete: SetNull)

      @@map("cash_transactions")
    }
    ```
- **Ação**: Executar `npx prisma db push` para aplicar a alteração local de forma segura ou criar migração.

#### 2. Baixa / Pagamento de Custo Fixo
- **Arquivo**: `src/modules/fixed-costs/domain/use-cases/pay-fixed-cost.use-case.ts`
- **Lógica**:
  - Recebe `fixedCostId`, `amount`, `cashRegisterId`, `description`.
  - Verifica se o Custo Fixo existe.
  - Verifica se o Caixa existe e se está ativo (caso contrário lança exceção).
  - Cria um novo registro em `CashTransaction` de tipo `"OUTFLOW"` vinculando ao caixa e ao custo fixo.
  - Retorna a transação criada.

### Onda 4: Controladores e Ajuste de Caixa (Controller)

#### 1. Controller de Custos Fixos
- **Arquivo**: `src/modules/fixed-costs/infrastructure/controllers/fixed-costs.controller.ts`
- **Endpoints**:
  - `POST /fixed-costs`: Criar Custo Fixo
  - `GET /fixed-costs`: Listar todos
  - `PUT /fixed-costs/:id`: Editar
  - `DELETE /fixed-costs/:id`: Excluir
  - `POST /fixed-costs/:id/pay`: Pagar Custo Fixo (lançar saída em caixa ativo)

#### 2. Atualização do Resumo do Caixa (`GetCashRegisterSummaryUseCase`)
- **Arquivo**: `src/modules/cash-registers/domain/use-cases/get-cash-register-summary.use-case.ts`
- **Modificação**:
  - Carregar todas as `CashTransaction` vinculadas ao caixa.
  - Somar entradas manuais (`ENTRY`) em `totalEntries`.
  - Somar saídas manuais e de custos fixos (`OUTFLOW`) em `totalOutflows`.
  - Reajustar o faturamento bruto: `totalGross = totalReceived (de pedidos) + totalEntries`.
  - Reajustar o faturamento líquido: `totalNet = totalGross - totalCardFees - totalOutflows`.
  - Incluir no payload final:
    ```json
    summary: {
      ...,
      totalGross,
      totalCardFees,
      totalOutflows,
      totalEntries,
      totalNet
    },
    transactions // Listagem detalhada das transações
    ```
